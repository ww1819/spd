# 备货出库 · 引用备货验收单 — 业务需求规格

> **状态**：定稿（待开发）  
> **范围**：高值 **备货出库**（`gz_shipment`，前端页 `spd-ui/src/views/gzOrder/audit`）  
> **关联**：备货验收（`gz_order`，`order_type = 101`）  
> **不在范围**：备货退货(TH)、备货退库(TK) 的引用规则（引用状态单独统计，互不干扰）

---

## 1. 业务目标

出库制单时，可**一次性**从已审核的**备货验收单**按当前出库仓库带入条码明细，并确定出库科室；通过引用状态、条码唯一占用、仓库/科室锁定，避免串库串科、重复出库，以及与退库/退货引用混淆。

---

## 2. 制单方式（互斥，V1）

| 方式 | 说明 |
|------|------|
| **A. 引用验收单** | 明细**仅**来自**一次**引用确认；绑定一张验收单后不可换单、不可再开引用 |
| **B. 扫码 / 手工添加** | 不引用验收单，通过扫码或「添加」维护明细 |

### 2.1 互斥规则

- 明细**为空**时：可选用 A 或 B 其中一种开始制单。
- **一旦执行引用带入**（方式 A）：本单进入「引用模式」，禁止扫码、禁止手工添加、禁止再次引用。
- **一旦已有明细**（含扫码/手工，方式 B）：禁止打开「引用验收单」。
- **一张出库单最多引用一张验收单**；引用只能在**明细为空且尚未绑定验收单**时发生。

### 2.2 引用模式（方案 A，最严）

引用确认并成功写入明细后：

- 明细**只能**来自该次引用结果（确认时已过滤无库存条码）。
- **不允许任何追加**：禁扫码、禁手工「添加」、禁再次「引用验收单」。

---

## 3. 表头字段与锁定

| 字段 | 规则 |
|------|------|
| **仓库** | 必选。未选仓库时：禁用「引用验收单」；扫码不可用 |
| **科室** | 出库必填（保存/审核校验） |

### 3.1 锁定时机

- **引用确认**并成功写入明细后 → 立即锁定仓库、科室。
- **扫码**成功写入明细后 → 立即锁定仓库、科室。
- 锁定后不可修改仓库、科室（已审核单据只读）。

### 3.2 扫码前置

- 仓库、科室为空时：院内码扫描框**禁用**，提示先选仓库、科室。
- 引用模式下：扫码**始终禁用**。

---

## 4. 引用验收单 — 交互流程

### 4.1 打开引用窗条件

同时满足：

1. 表头**已选出库仓库**；
2. 出库单**明细为空**；
3. 本单**未绑定** `refAcceptanceId`（从未成功完成引用）。

不满足时：「引用验收单」按钮禁用或点击提示原因。

### 4.2 引用窗筛选条件

| 条件 | 规则 |
|------|------|
| **仓库** | 固定为表头已选出库仓库 |
| **科室** | 表头**已选科室** → 条件**锁定**为该科室，列表**仅**展示 `applyDepartmentId` 等于该科室的验收单；表头**未选科室** → 科室条件为空，不按科室过滤 |
| **引用状态 Tab** | **未引用** / **部分引用** / **已引用**（见第 6 节） |
| **单据范围** | 仅 **已审核** 备货验收单（`order_type = 101`，`order_status = 2`） |

列表建议展示：验收单号、申请科室、制单/审核时间、可引条码数等。

### 4.3 验收单科室字段

- **统一使用** `applyDepartmentId`（申请科室）。
- 引用窗筛选、出库科室回填、与表头科室联动，均按 `applyDepartmentId`。
- **不使用** `departmentId` 作为验收单科室（除非产品后续另行规定）。

### 4.4 选择验收单并确认

1. 用户**单选**一条验收单。
2. 点击「确定」后确定**出库科室**：
   - 表头**已有科室**（引用窗已按该科室筛选）→ 出库科室 = 表头科室；
   - 表头**无科室** → 出库科室 = 验收单 `applyDepartmentId`；
   - 验收单 `applyDepartmentId` 也为空 → **弹窗**让用户选择出库科室 → 写入表头。
3. 写入表头科室后 → **锁定科室**。
4. 系统按「表头仓库 + 所选验收单」加载可引条码行。

### 4.5 无库存条码处理

- 按当前**出库仓库**校验每条验收条码在 `gz_depot_inventory` 是否 `qty > 0`。
- **有库存** → 写入出库明细。
- **无库存** → **不写入**；弹窗列出**缺失院内码**（可含耗材名称），说明「以下条码在当前仓库无库存，不会带入」。
- **全部无库存** → 引用失败，不写入明细，不写 `refAcceptanceId`，不进入引用模式（不改变仓/科锁定状态）。
- **部分有库存** → 用户确认后仅带入有库存行，写入 `refAcceptanceId`，进入引用模式并锁定仓/科。

### 4.6 引用完成后

- 出库单绑定 `refAcceptanceId`（所选验收单 ID），**生命周期内不可变更**。
- 立即锁定仓库、科室。
- 禁用：扫码、手工「添加」、再次「引用验收单」。

### 4.7 清空明细与换单

- 用户可删除明细行，但**清空明细后**：
  - **不**解除 `refAcceptanceId` 绑定；
  - **不允许**再打开引用（含同一张或**换另一张**验收单）；
  - **不允许**扫码/手工追加（引用模式仍有效）。
- 若要引用**另一张**验收单或重新引用：须**删除整张出库单**后新建。

---

## 5. 扫码 / 手工添加（非引用路径）

| 规则 | 说明 |
|------|------|
| 前置 | 仓库、科室均已选择 |
| 与引用互斥 | 已有明细 → 不可再引用；已绑定 `refAcceptanceId` → 不可扫码/添加 |
| 锁定 | 首次扫码或添加写入明细后，锁定仓库、科室 |

扫码：按「院内码 + 出库仓库」查备货库存（与现有 `getDepotByInHospitalCodeForOutbound` 一致）。

---

## 6. 出库引用状态（验收单侧）

### 6.1 与退库/退货分离

- **出库引用**单独统计，不计入备货退货(TH)、备货退库(TK) 的引用。
- 退库/退货继续使用 `gz_refund_goods_entry_ref` 等表。

### 6.2 存储设计

| 层级 | 说明 |
|------|------|
| **条码级真相** | `gz_order_entry_code_ref`（主键及外键均为 `varchar(36)` / UUID7），`tgt_bill_kind = 'GZ_SHIPMENT'` |
| **验收单冗余** | `gz_order.shipment_ref_status`（仅 `order_type = 101`） |

**`shipment_ref_status` 取值：**

| 值 | 含义 |
|----|------|
| `0` | 未引用 |
| `1` | 部分引用 |
| `2` | 全部引用 |

### 6.3 统计粒度

- 基准：验收单下有效**条码行**（`gz_order_entry_inhospitalcode_list`）。
- **已占用**：存在于 `gz_order_entry_code_ref` 且 `tgt_bill_kind = 'GZ_SHIPMENT'`（**含草稿出库单**）。
- **仍可引用**：未被上述占用，且在指定仓库仍有备货库存（`gz_depot_inventory.qty > 0`）。

### 6.4 验收单级状态定义

| 状态 | 定义 |
|------|------|
| **未引用** | 无任何条码被出库引用 |
| **部分引用** | 至少 1 条已引用，且仍有可引条码 |
| **已引用** | 所有可出库条码均已引用，或无剩余可引条码 |

### 6.5 引用窗 Tab

- 三个 Sheet：**未引用** / **部分引用** / **已引用**。
- 与仓库、科室（锁定）条件**组合**筛选。

### 6.6 释放与回写

| 事件 | 处理 |
|------|------|
| **未审核出库单删除** | 删除本单 `gz_order_entry_code_ref`（`GZ_SHIPMENT`）及 `gz_shipment_entry_ref` → **重算**涉及验收单的 `shipment_ref_status` |
| **出库单保存/更新** | 维护引用关系 → 重算相关验收单冗余状态 |
| **出库单作废** | **本期无作废入口**，不做 |

**说明（V1）**：删除即释放引用占用**足够**。已审核出库单通常不可删除，引用关系保留作追溯；**仅未审核**出库单删除时释放条码占用并重算验收单状态。

---

## 7. 条码唯一引用（强约束）

- 同一验收条码明细（`src_barcode_line_id`）在 **出库维度**（`GZ_SHIPMENT`）**全局仅可引用一次**。
- 不区分草稿/已审核：已被任一出库单占用则不可再引。
- 建议：唯一索引 `(src_barcode_line_id, tgt_bill_kind)`，`tgt_bill_kind = 'GZ_SHIPMENT'`。
- 保存出库单时后端校验；冲突提示含占用单号/院内码。
- **未审核**占用方出库单删除后释放，该条码可再次被引。

---

## 8. 一张出库单 ↔ 一张验收单

- 每张备货出库单**最多**关联 **1 张**备货验收单。
- 表头冗余 `refAcceptanceId`（`gz_shipment.ref_acceptance_id`，`varchar(36)`；或明细 `refSrcAcceptanceId` 归一校验）。
- 绑定后不可更换；清空明细不可换单重引。

---

## 9. 后端校验要点

| 校验项 | 说明 |
|--------|------|
| 仓库、科室 | 出库单必填 |
| 引用模式 | 明细均来自同一 `refAcceptanceId`；禁止混入非引用来源行 |
| 条码唯一 | 保存前检查 `src_barcode_line_id` 未被其他出库单占用 |
| 库存 | 沿用 `GzStockValidationService.assertShipmentOutbound`；引用确认时的无库存提示为前端体验层，后端仍强校验 |
| 引用关系 | 保存写 `gz_order_entry_code_ref`、`gz_shipment_entry_ref`；更新先删后写；**未审核删除**清理并重算 |

## 11. UI 行为汇总

| 操作 | 条件 | 结果 |
|------|------|------|
| 引用验收单 | 已选仓库 && 明细为空 && 未绑定 `refAcceptanceId` | 打开引用窗 |
| 引用确定 | 单选一条验收单 | 定科室 → 带明细 / 提示无库存 |
| 引用成功 | — | 写 `refAcceptanceId`；锁定仓/科；禁扫码、禁添加、禁再引用 |
| 清空明细 | 曾引用成功 | 仍绑定 `refAcceptanceId`；仍禁引用、禁扫码、禁添加 |
| 扫码 / 添加 | 未绑定验收单 && 非引用模式 && 已选仓/科 | 可用；首次写入后锁定 |
| 改仓库/科室 | 无明细 && 未锁定 && 未绑定验收单 | 允许；否则禁止 |
